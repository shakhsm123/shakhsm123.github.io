DeiT: Data-efficient Image Transformers from Scratch

A transformer that learns to see by standing on the shoulders of a CNN — built cell by cell, from patch tokenization to distillation-driven convergence.
~3k tokens · estimated 12 min read

Introduction

A learning agent whose only teacher is a past version of something smarter than itself. DeiT — Data-efficient Image Transformers — is exactly this: a pure attention model that learns to classify images not just from labeled data, but by internalizing the representations of a convolutional network that has already mastered the task. The animation below captures the essence of this process.

The teacher (ResNet, blue) pulses with learned visual knowledge — convolutional priors about locality, texture, inter-class similarity. Energy streams flow to the student (DeiT, orange) during training, shaping its representations without ever seeing ImageNet.

The original Vision Transformer required JFT-300M — Google's private 300 million image dataset — to outperform CNNs. On ImageNet alone it badly underperformed ResNets. DeiT matched that performance using only ImageNet, by training with a ResNet teacher and a dedicated distillation token. This post implements DeiT from scratch on CIFAR-10, traces every design decision, and shows the live training dynamics.

The problem with ViT on small data

CNNs embed two powerful priors directly into their structure. Transformers have neither. The animation below makes the difference visceral.

Inductive biases: what CNNs know that ViTs don't

A convolutional kernel can only see a local neighbourhood at a time — 3×3 or 5×5 pixels. The same kernel is applied everywhere (weight sharing), so a feature detector learned at one location works everywhere else. Transformers make no such assumption — every patch attends to every other from layer one.

CNN
local kernel · weight sharing
translation invariance built in
3×3 kernel — sees only its neighbourhood
ViT
global attention · no locality prior
learns spatial structure from data
every patch attends to all others
CNN kernel sweeps the dog image locally — 9 of 49 patches visible per step, hard boundary shown. ViT selected patch draws attention lines to all 48 others simultaneously. On 50,000 CIFAR-10 images, learning this from scratch is a severe disadvantage.
The DeiT insight. You cannot give a transformer locality bias by changing its architecture. But you can give it the outcome of that bias — transfer the CNN's learned representations via knowledge distillation.

Knowledge distillation

Train a large accurate teacher first, then train the student to produce the same outputs — not just the same labels. The key mechanism is soft probability vectors: rather than a one-hot [0,0,1,0,...] for "cat," the teacher produces a distribution over all classes that encodes relational knowledge.

Temperature — how much the teacher reveals

Temperature τ divides logits before softmax, flattening the distribution. At τ=1 the teacher output is nearly one-hot. At τ=3 inter-class similarity emerges. Drag the slider below.

τ 1.0 ← sharp (label only) · · · · soft (relational) →
At τ=1 the student learns "cat, full stop." At τ=3 it also learns "somewhat like a dog." Right panel tracks what types of knowledge accumulate in the DIST token as temperature rises.

Teacher: ResNet50 adapted for 32×32

ResNet50's first conv and maxpool would reduce a 32×32 image to 4×4 before the first residual block. Two lines fix this: 

teacher.conv1   = nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1, bias=False)
teacher.maxpool = nn.Identity()
teacher.fc      = nn.Linear(teacher.fc.in_features, num_classes)

The teacher reaches 95.0% accuracy after 20 epochs of full fine-tuning with differential learning rates.

Architecture

Patch embedding

A 32×32 image is divided into 4×4 non-overlapping patches → 64 tokens each of raw dimension 48 → projected to model dimension D=192:

zpatches = Xpatches · WE,   WE ∈ ℝ48×192 Conv2d(in=3, out=192, kernel=4, stride=4) — tiles and projects in one pass
class PatchEmbedding(nn.Module):
  def __init__(self, in_channels, embed_dim, patch_size, stride):
    super().__init__()
    self.projection  = nn.Conv2d(in_channels, embed_dim, patch_size, stride=patch_size)
    self.cls_token   = nn.Parameter(torch.randn(1, 1, embed_dim) * 0.2)
    self.dist_token  = nn.Parameter(torch.randn(1, 1, embed_dim) * 0.2)
    self.pos_embedding = nn.Parameter(torch.randn(1, num_patches+2, embed_dim) * 0.2)

  def forward(self, x):
    B = x.shape[0]
    x = self.projection(x).flatten(2).transpose(-1, -2)   # (B, 64, 192)
    cls_tokens  = self.cls_token.expand(B, -1, -1)
    dist_tokens = self.dist_token.expand(B, -1, -1)
    x = torch.cat([cls_tokens, dist_tokens, x], dim=1)    # (B, 66, 192)
    return x + self.pos_embedding

Multi-head self-attention

Attention(Q, K, V) = softmax(QK⊤ / √dk)V d_k = 192/3 = 64 — scaling prevents softmax saturation
class MHA(nn.Module):
  def __init__(self, embedding_dim, num_heads):
    super().__init__()
    self.num_heads = num_heads
    self.d_k       = embedding_dim // num_heads
    self.scale     = math.sqrt(self.d_k)
    self.qkv       = nn.Linear(embedding_dim, embedding_dim*3, bias=False)
    self.proj      = nn.Linear(embedding_dim, embedding_dim)
    self.attn_dropout = nn.Dropout(0.1)
    self.proj_dropout = nn.Dropout(0.1)

  def forward(self, x):
    qkv = self.qkv(x)
    q, k, v = torch.chunk(qkv, 3, dim=-1)
    B, N, D = q.shape
    q = q.reshape(B, N, self.num_heads, self.d_k).transpose(1, 2)
    k = k.reshape(B, N, self.num_heads, self.d_k).transpose(1, 2)
    v = v.reshape(B, N, self.num_heads, self.d_k).transpose(1, 2)
    score = torch.matmul(q, k.transpose(-1, -2)) / self.scale
    attn  = self.attn_dropout(torch.softmax(score, dim=-1))
    out   = torch.matmul(attn, v).transpose(1,2).reshape(B, N, D)
    return self.proj_dropout(self.proj(out))

CLS and DIST — two special tokens

sequence: [CLS] [DIST] [p1] [p2] ··· [p64] 0 1 2 3 ··· 65 ŷcls = Wcls · z[0] → ℝ10 # supervised by ground-truth ŷdist = Wdist · z[1] → ℝ10 # supervised by teacher inference: prediction = argmax( (ŷ_cls + ŷ_dist) / 2 )
class DeiT(nn.Module):
  def __init__(self, image_size, in_channels, embedding_dim, num_heads,
               num_layers, mlp_ratio, patch_size, num_classes, dropout):
    super().__init__()
    self.patch_embedding = PatchEmbedding(in_channels, embedding_dim, patch_size, stride=patch_size)
    self.transformer = nn.Sequential(*[TransformerBlock(embedding_dim, num_heads, mlp_ratio)
                                       for _ in range(num_layers)])
    self.norm      = nn.LayerNorm(embedding_dim)
    self.cls_head  = nn.Linear(embedding_dim, num_classes)
    self.dist_head = nn.Linear(embedding_dim, num_classes)
    self.dropout   = nn.Dropout(dropout)

  def forward(self, x):
    x = self.dropout(self.patch_embedding(x))
    x = self.norm(self.transformer(x))
    cls_logits  = self.cls_head(x[:, 0, :])
    dist_logits = self.dist_head(x[:, 1, :])
    if self.training:
      return cls_logits, dist_logits
    return (cls_logits + dist_logits) / 2

Loss function — two signals, one model

The animation below cycles automatically through three phases: CLS path lights up, then DIST path, then both combine. The formulas are always visible — only the active one is fully lit.

CLS token · cross-entropy loss active
CLS token (blue) supervised by hard ground-truth labels via cross-entropy. DIST token (orange) supervised by the teacher's soft probability vector via KL divergence. Both feed into L_total with equal weight α=0.5.

Classification loss

ℒcls = CrossEntropy(ŷcls, y) = −log softmax(ŷcls)[y] = −ŷcls[y] + log ∑j exp(ŷcls[j])
  cls_loss = nn.CrossEntropyLoss()(cls_logits, labels)

Distillation loss

The τ² factor compensates for gradient magnitude reduction — without it distillation gradients are τ² times smaller than classification gradients, breaking the intended balance.

ℒdist = τ² · KL( log_softmax(ŷdist/τ) ‖ softmax(ŷteacher/τ) ) = τ² · ∑j Pteacher(j) · log [ Pteacher(j) / Qdist(j) ] P = softmax(ŷ_teacher/τ), Q = softmax(ŷ_dist/τ), τ = 3 τ² correction: ∂softmax(z/τ)/∂z scales by 1/τ², multiply back to restore scale
  dist_loss = temperature**2 * F.kl_div(
      F.log_softmax(dist_logits  / temperature, dim=-1),
      F.softmax(teacher_logits   / temperature, dim=-1),
      reduction='batchmean'
  )

Combined loss

ℒtotal = (1−α) · ℒcls + α · ℒdist = 0.5 · ℒcls + 0.5 · ℒdist # α = 0.5 α → 0 : plain ViT, no distillation α → 1 : purely mimics teacher, ignores ground-truth
  combined_loss = alpha * cls_loss + (1 - alpha) * dist_loss
  return (combined_loss, cls_loss, dist_loss)

Training

warmup (epochs 1–5): lr(e) = LR · e / 5 cosine (epochs 6–50): lr(e) = LR/2 · (1 + cos(π(e−5)/(E−5))) LR = 1e-4 · weight_decay = 0.05 · batch_size = 128
warmup_scheduler = torch.optim.lr_scheduler.LinearLR(
    optimizer, start_factor=1e-6, end_factor=1.0, total_iters=warmup_epochs)
cosine_scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=epochs - warmup_epochs)
scheduler = torch.optim.lr_scheduler.SequentialLR(
    optimizer, schedulers=[warmup_scheduler, cosine_scheduler], milestones=[warmup_epochs])
